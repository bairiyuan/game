# 单机游戏-打小怪 (Desktop Pet: Monster Shooter)

## 一、 介绍

这是一个轻量级的纵向射击类游戏玩法。用户通过控制桌宠在屏幕底端左右移动，向上方射击以击退不断掉落的小怪兽 。通过击败不同等级的怪兽收集分数，获得挑战自我的成就感。

## 三、 UX设计（参考示例游戏打飞机）

游戏逻辑流程如下 ：

1. **准备阶段**：点击开始进入战斗。
2. **战斗阶段**：通过鼠标拖动移动自动射击消灭怪兽；怪兽碰撞桌宠则触发结束判定。
3. **结算阶段**：展示成绩。

### (三) 游戏进程与生成逻辑

游戏进程与生成逻辑 

#### 1. 障碍（怪兽）生成机制

怪兽采用基于时间密度的**动态随机生成**逻辑：

- **生成位置**：怪兽从屏幕上方（Y=0）随机 X 轴坐标点生成。

- **种类抽取**：每次生成时从“大、中、小”三种怪兽库中随机抽取。

- **频率递增**：初始频率为每秒生成 **1** 只；随着游戏进行，最终频率增加至每秒 **5** 只 。

#### 2. 速度与距离单位说明 

- **距离单位**：像素 (**px**) 。

- **速度单位**：像素/秒 (**px/s**) 。

- **基础参照**：屏幕横向宽度为 **2560** 像素 。

#### 3.速度增长模型 

游戏难度随时间均匀加速，怪兽下落速度不断提升：

- **最低速度**：v(min) = 150px / s 。

- **最高速度**：v(max) = 800px / s 。

- **加速规则**：
- **核心规则**：每生存 **30** 秒，怪兽的基础下落速度提升 **50px / s** 。
  
- **数量增长**：每 **30** 秒后每秒出现在屏幕上的怪兽上限**+1**。
